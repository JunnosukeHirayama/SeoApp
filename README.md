## Overview (概要)

WEBメディアのSEO業務における記事の内部リンク管理を効率化するためのアプリです。具体的にはどの記事がどの記事へ内部リンクが繋がっているのかや、
記事ごとの保有リンクの数、SEOで重要になるトピッククラスターの管理を視覚的に分かりやすくし、内部リンク情報の追加削除も直感的に行えるようにするためのものです。

※本プロジェクトは学部生の際にローカル環境で開発し、ポートフォリオとして本リポジトリにアーカイブしたものです。

## Motivation (開発の動機)

以前、AI関係のWEBメディアにおけるSEO業務での長期インターンシップに参加していました。そこでの内部リンク管理はスプレッドシートを使ったもので、記事数が膨大だったため、
視覚的に分かりづらく、記事ごとのトピッククラスター管理も大変で手間がかかっていたため、インターン生全員の業務効率を上げるために制作しました。

## Tech Stack (使用技術)

* **Languages:** JavaScript / HTML
* **Library:** Google Apps Script API (SpreadsheetApp)
* **Infrastructure & Tools:** Google Workspace(Sheets as a Database) / Google Apps Script Editor

## Key Features & Technical Challenges (主な機能と技術的な工夫・苦労した点)

### 主な機能
* 内部リンク情報の追加・削除
* 指定した記事の内部リンク数・内部リンク先の一覧表示
* 内部リンク数ランキング表示
* トピッククラスター作成
* 各親記事のトピッククラスター表示
* 新規記事追加

### 技術的な工夫・解決した課題
* **課題1:** スプレッドシートには標準的なRDBのような外部キー制約がないため、データの削除時に「参照元だけ残る（データが浮く）」問題が発生する。
    * **解決策:** deleteArticle 関数において、記事削除時に関連する Links（内部リンク）、Members（クラスター所属）、Clusters（ピラー指定）の全てのシートから該当IDを検索・削除・更新するロジックを実装し、データの整合性（参照整合性）を維持する仕組みを構築した。


## Installation & Usage (環境構築・実行方法)

本アプリケーションは Google Apps Script (GAS) を使用して動作します。以下の手順でご自身の Google アカウント環境にデプロイして実行可能です。

### 1. スプレッドシートの準備（データベース設定）

1. 新規に Google スプレッドシートを作成します。
2. 以下の4つのシートを作成し、1行目にそれぞれ指定のヘッダー（見出し）を入力してください。
   * **Articles**: `ID`, `タイトル`, `URL`
   * **Links**: `SourceID`, `TargetID`
   * **Clusters**: `ClusterID`, `クラスター名`, `PillarID`
   * **ClusterMembers**: `ClusterID`, `ArticleID`

### 2. スクリプトの配置

1. 作成したスプレッドシートの上部メニューから **「拡張機能」>「Apps Script」** を開きます。
2. デフォルトの `コード.gs` に、本リポジトリのバックエンド用コード（`.gs` ファイルの内容）をコピー＆ペーストします。
3. エディタ左側の「ファイル」横にある「＋」ボタンを押し、**「HTML」** を選択して `index` という名前でファイルを作成します（拡張子は自動で `.html` になります）。
4. 作成した `index.html` に、本リポジトリの `index.html` の内容をコピー＆ペーストして保存します。

### 3. デプロイ（Webアプリとしての公開）

1. スクリプトエディタ画面右上の **「デプロイ」>「新しいデプロイ」** をクリックします。
2. 左側の歯車アイコン（種類の選択）をクリックし、**「ウェブアプリ」** を選択します。
3. 以下の設定を行い、「デプロイ」ボタンを押します。
   * **次のユーザーとして実行:** `自分`
   * **アクセスできるユーザー:** `自分のみ`（共有する場合は適宜変更してください）
4. ※初回実行時はアクセス承認のポップアップが出ます。「詳細を表示」→「安全ではないページへ移動」から権限を許可してください。
5. 発行された **「ウェブアプリのURL」** にブラウザでアクセスすると、アプリケーションのUI画面が表示されます。
